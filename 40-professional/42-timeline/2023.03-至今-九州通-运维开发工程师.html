      河南九州通医药有限公司｜高级系统工程师（基础设施架构 / 自动化 / 安全合规）

      时间：2023 年 3 月–至今（2025 年 10 月，约 2.5 年）

      任职概览

      • 职级定位：高级系统工程师（Infrastructure Lead），直接向部门经理汇报

      • 核心职责：在集团多林多域的复杂混合架构下，负责省公司及 6 个地市分公司的 IT 基础设施全生命周期治理。

      • 技术关键词：Hybrid Virtualization (KVM/Hyper-V) · Windows Server Core · AD Forest Trust · Zero Trust (SSH CA) · Infrastructure as Code (Ansible)

      • 业务价值：在医药流通行业严格的合规要求（等保/GSP）下，通过技术架构升级，实现了成本（TCO ↓40%）、安全（审计 100%）与可用性（SLA 99.9%）的三角平衡。

      核心职责边界与技术深度

      1. 异构混合基础设施 (Heterogeneous Infrastructure)

      • 双栈虚拟化底座：设计并维护 KVM (Linux Workloads) 与 Hyper-V (Windows Workloads) 的混合集群。针对关键业务场景，大规模部署 Windows Server Core 版本，显著降低资源占用并缩减 60%+ 的攻击面。

      • 网络高可用架构：构建基于 OSPF/BGP 动态路由协议的网络层，结合 VIP (Keepalived/LVS) 实现控制面与数据面的高可用；设计 ” 双专线 + 5G 应急 ” 的三重链路冗余机制，保障极端灾难下的业务连续性。

      2. 身份治理与零信任体系 (Identity & Zero Trust)

      • 企业级身份编织：维护本地 AD 域与集团 AD 林的 双向信任关系 (Two-way Forest Trust)。利用 LDAP 作为统一身份源（Source of Truth），打通 JumpServer、VPN 及业务系统，实现 SSO 单点登录与权限的生命周期自动化管理。

      • 零信任准入控制：构建基于 JumpServer + SSH CA 的零信任访问体系。摒弃静态密码与分散公钥，通过 Ansible 自动化维护主机信任锚点 (Trust Anchor)，实现资产访问的 ” 一次一密 ” 与全链路审计。

      3. 全栈可观测性与合规 (Observability & Compliance)

      • 立体监控体系：集成 Zabbix (底层硬件/网络)、Prometheus (中间件/数据库) 与 ELK (日志关联分析)。通过告警抑制与分级策略，消除告警风暴，实现故障 3 分钟发现 / 10 分钟响应。

      • 合规闭环：主导网络安全等级保护（二级）建设，建立 ” 技术（日志审计/备份加密）+ 管理（SOP/台账）” 的双模合规体系，确保通过测评及年度审计。

      关键项目实战（精选）

      项目 5：混合云架构下的全链路零信任与统一监控体系

      时间：2025.03–2025.06（4 个月）

      角色：项目架构师 / 落地负责人

      STAR 深度解析

      S（背景/复杂性）

      • 架构痛点：环境处于集团与本地的 ” 多林多域 ” 混合信任边界，存在 Windows/Linux 异构资产共存、特权账号管理混乱、公钥散落难以审计等问题。

      • 业务压力：需在满足审计合规（所有操作可追溯）的同时，不降低运维效率。

      T（目标/KPI）

      • 零信任落地：构建以 ” 身份 ” 为边界的访问控制体系，实现 100% 资产纳管与会话录像。

      • 运维自动化：利用 IaC (Infrastructure as Code) 思路，将新资产纳管时间缩短至秒级。

      • 底座高可用：确保认证中心与监控中心具备网络层自动容灾能力。

      A（关键架构决策）

      1. 身份源整合（Identity Fabric）：

        • 对接集团 AD 林，利用 LDAP 同步服务将组织架构映射至 JumpServer，实现人员入转调离权限的 JIT (Just-In-Time) 自动化流转。

      2. 构建 SSH CA 信任链：

        • 部署 JumpServer 作为 CA 签发机构。

        • 编写 Ansible Playbook，批量分发 TrustedUserCAKeys 配置至存量 Linux 资产，并固化至 OS 初始化镜像中，实现新机上线自动互信。

      3. 网络层高可用设计：

        • 在核心网关层部署 VIP，结合路由策略，确保监控数据流与管理流量在物理链路切换时（如主专线中断切换至 5G）自动重路由，保障管理平面不失联。

      4. 告警治理（AIOps 初探）：

        • 在 Zabbix/Prometheus 上层接入统一事件中心，实施 告警收敛（Deduplication）与 依赖抑制（Suppression），解决 ” 网络抖动导致万台服务器同时报警 ” 的噪音问题。

      R（量化结果）

      • 安全质变：回收所有散落的 SSH Key 与 Root 密码，实现 100% 运维操作实名制与可回溯。

      • 效率飞跃：通过 Ansible + CA，资产纳管效率提升 90%，彻底消除因密钥管理不当导致的服务中断。

      • 稳定性承诺：基础设施监控与认证服务达到 99.9% 可用性。

      项目 4：虚拟化架构升级（VMware → KVM/Windows 混合栈）

      时间：2024.03–2024.10（8 个月）

      角色：方案设计者 / 迁移实施负责人

      STAR 深度解析

      S（背景/约束）

      • 成本与依赖：集团要求去商用化（下线 VMware），但核心业务（ERP/WMS）深度依赖 Windows AD 特性，且对稳定性要求极高。

      • 技术挑战：需在异构虚拟化平台间迁移 50TB+ 数据，且需保证 RTO (恢复时间目标) < 4 小时。

      T（目标/KPI）

      • 构建 KVM (计算型) + Hyper-V (依赖型) 的混合集群，降低 TCO 40%。

      • 实现计算资源的超分复用（Overcommitment）与 IO 隔离。

      A（关键架构决策）

      1. 异构选型策略：

        • 核心组件 (AD/DNS/GPO)：迁移至 Windows Server Core (Hyper-V)，利用原生兼容性保障身份服务稳定性，同时利用 Core 模式减少补丁更新频率。

        • 通用业务 (Web/App/DB)：迁移至 KVM (CentOS/Ubuntu)，利用 Virtio 驱动优化 I/O 性能。

      2. 平滑迁移流程：

        • 设计 ” 在线数据同步 + 离线最终切换 ” 方案，利用中间存储过渡，将关键业务停机窗口控制在分钟级。

      3. 性能调优：

        • 在 KVM 侧实施 CPU Pinning (绑核) 与 HugePages (大页内存) 优化，确保数据库业务性能不低于物理机基准的 95%。

      R（量化结果）

      • 成功构建混合虚拟化集群，宿主资源利用率提升 40%，节省每年 VMware 授权费用数十万元。

      • 建立了基于 Libvirt/Virsh 的标准化命令行管理规范，纳入自动化运维体系。

      项目 1：网络安全等级保护（二级）整改与终端标准化治理

      时间：2023.05–2023.07（3 个月）

      角色：技术负责人

      (保留核心逻辑，强化管理维度) 核心亮点：

      • 终端标准化：利用 AD Group Policy (GPO) 替代人工运维，批量下发安全基线（如 USB 管控、屏幕锁定、密码复杂度），统一全省 1000+ 终端的安全水位。

      • 合规双模体系：构建了 ” 技术（防火墙/堡垒机/加密）+ 管理（SOP/冷备/演练）” 的合规闭环，确保合规不仅是应对检查，而是内化为日常运维动作。

      技能矩阵 (Skill Matrix)

      领域核心技能栈
      混合基础设施KVM, Windows Server Core, Hyper-V, OSPF/BGP, VIP/Keepalived, 物理链路冗余设计
      身份与零信任Active Directory (多林信任), LDAP, GPO, JumpServer, SSH CA, Kerberos
      自动化与代码Ansible (Playbook), PowerShell, Shell Scripting, Git
      可观测性Zabbix, ELK Stack, Prometheus, Grafana, AIOps (告警治理)
      合规与治理等保 2.0 (二级), IT 资产全生命周期管理, RTO/RPO 灾备规划

      核心竞争力总结 (Why Me?)

      1. 复杂架构驾驭者：具备在集团级 多林多域 环境下，统筹管理 Windows (AD/Core) 与 Linux (KVM) 异构栈的实战经验，能解决跨域信任与资源整合难题。

      2. 安全内建设计 (Security by Design)：不依赖外挂式安全，而是通过 SSH CA、Server Core、GPO 等原生机制构建内生安全体系。

      3. 高可用落地专家：从物理链路的 OSPF 到应用层的 VIP，具备全栈高可用设计能力，真正实现业务的 ” 永不掉线 “。
